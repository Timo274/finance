# Salary Allocation Planner / Capital Queue

Минималистичное web-приложение для **планирования будущей зарплаты** — не трекер расходов, а
«personal finance cockpit». Помогает заранее разложить доход по датам, приоритетам, желаниям,
обязательствам и долгосрочным целям ещё до того, как деньги пришли.

> Сделано под профиль: студент, доход ~25 000 грн/мес, живёт с родителями, низкие обязательные
> расходы, много желаний и потребность расставить их в очередь.

## Что умеет

- **Кабинет** — зарплата и стабильные расходники: обязательные расходы, страховка и инвестиции.
  В желания, кошельки и ручной план идут только излишки после этих пунктов.
- **Очередь желаний (Capital Queue)** — единый мастер-список желаний, который переносится из
  месяца в месяц. У каждого: стоимость, категория, приоритет 1–5, тип (must/should/nice),
  дедлайн, «не раньше даты», можно ли отложить, эмоциональное желание, долгосрочная ценность,
  накопленная сумма и фильтры.
- **Кошельки месяца** — карманы под конкретный месяц: сумма, назначение, пропорции и прогресс
  накоплений на желания.
- **Инвестиции** — отдельная вкладка для monthly update: куда и сколько вложено, график и пропорции.
- **План распределения** — ручное распределение излишков по желаниям плюс авто-подсказка:
  дедлайны → высокий приоритет и долгосрочная ценность → остальное.
- **Сценарии** — Сбалансированный / Больше копить / Карьерный капитал / Наслаждаться жизнью /
  свой сценарий. Сравнение по слоям, стабильным пунктам и остатку излишков.
- **Trade-off** — для любой покупки: «останется X», «вытеснит …», «не помещается в излишки».
- **История решений** — закрытые месяцы со снимком: что куплено, что отложено, остаток.
- **AI-ассистент** *(опционально)* — советует, что купить первым и что отложить. Без ключа
  приложение работает полностью, ассистент просто выключен.

## Стек

- Backend: **Node.js + Express + SQLite** (`better-sqlite3`)
- Frontend: один SPA на чистом HTML/CSS/JS (тёмно-синий минималистичный дизайн)
- Auth: один пользователь + PIN (bcrypt + подписанный cookie-токен)

## Запуск локально

```bash
npm install
cp .env.example .env   # при желании отредактируйте
npm start              # http://localhost:3000
npm test               # unit-тесты allocation engine
```

При первом открытии приложение попросит **создать PIN**. В production можно дополнительно задать
`SETUP_TOKEN`, тогда первый PIN нельзя будет создать без этого токена.

## Переменные окружения

См. `.env.example`. Ключевые:

| Переменная | Назначение |
|---|---|
| `PORT` | порт сервера (по умолчанию 3000) |
| `SESSION_SECRET` | секрет для подписи сессий (**обязателен в production**) |
| `SETUP_TOKEN` | опциональный токен для первого создания PIN на пустой базе |
| `DB_PATH` | путь к файлу SQLite (по умолчанию `./data/app.db`) |
| `AI_PROVIDER` | `openai` \| `anthropic` \| `gemini` (опционально) |
| `AI_API_KEY` | ключ выбранного провайдера (опционально) |
| `AI_MODEL` | переопределить модель (опционально) |
| `BACKUP_ENABLED` | `false`, чтобы выключить автоснапшоты JSON |
| `BACKUP_DIR` | каталог JSON backup (по умолчанию рядом с SQLite, на Fly это `/data/backups`) |
| `BACKUP_INTERVAL_HOURS` | период автоснапшота, по умолчанию `24` |
| `BACKUP_RETENTION` | сколько JSON backup хранить, по умолчанию `14` |

## Backup / restore

Экспорт JSON (`Данные и цели → Экспорт JSON`) содержит планы, желания, кошельки, цели,
ручные решения и инвестиционный портфель. Импорт полного backup заменяет текущие пользовательские
данные, но не трогает PIN/настройки сессии; перед восстановлением UI просит подтверждение.

Сервер также пишет scheduled JSON snapshots в `BACKUP_DIR`: один при старте и дальше раз в
`BACKUP_INTERVAL_HOURS`. На Fly это persistent volume рядом с SQLite (`/data/backups`).

## Деплой

Репозиторий содержит `render.yaml` для деплоя на [Render](https://render.com) в один клик
(persistent disk для SQLite). Аналогично разворачивается на Railway / Fly.io — нужен только
Node 18+, постоянный диск под `data/` и переменные окружения выше.
