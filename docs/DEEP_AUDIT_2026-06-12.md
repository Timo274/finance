## 2. Новые находки (этой волной внесены или вскрыты живым прогоном)

### 2.1 🔴 High · UX/Onboarding · Подсказка PIN противоречит валидации
- **Где:** `public/app.js:611–612` — «Придумайте PIN (минимум 4 цифры)», при этом клиент (`app.js:637`) и сервер (`routes/auth.js:33`) требуют ≥ 6.
- **Почему:** регрессия фикса 16.1 — валидацию подняли до 6, подсказку забыли.
- **Влияние:** первый же экран нового пользователя: вводит 4 цифры, как сказано, → получает ошибку. Худшее место для противоречия — точка входа.
- **Решение:** заменить текст на «минимум 6 цифр» в обеих строках.
```js
// app.js:611-612
? "Введите SETUP_TOKEN из переменных окружения и придумайте PIN (минимум 6 цифр)."
: "Это персональное приложение. Придумайте PIN (минимум 6 цифр)."
```

### 2.2 🟠 Medium · Финданные · `clampNum` молча искажает ввод вместо отказа
- **Где:** `src/routes/investments.js:10–14` + `isoDateOrToday`.
- **Факт (проверено фаззингом):** `price: -500` → записана покупка по цене **0**; `price: 9e99` → молча кламп до **1e12** (и попало в портфель как 45 трлн грн); `date: "not-a-date"` → молча подставлено сегодня.
- **Почему:** для финансового журнала молчаливая коэрция = порча данных. Пользователь промахнулся клавишей — система зафиксировала бессмыслицу без сигнала.
- **Влияние:** мусор в портфеле/итогах, который потом не объяснить.
- **Решение:** ошибки вместо коэрции:
```js
function strictNum(v, field, res) {
  const n = Number(v);
  if (!Number.isFinite(n) || n < 0 || n > NUM_CAP) {
    res.status(400).json({ error: "validation_failed", field });
    return null;
  }
  return n;
}
// и для даты: невалидная → 400, а не todayISO()
```

### 2.3 🟡 Medium · UI/DataViz · Экстремальные суммы ломают вёрстку и читаемость
- **Факт (скриншоты):** «Стоимость портфеля 89 960 000 189 8…» — число **обрезано** в hero-карте; «Регулярный взнос 2 000 000 004 220 грн / 6 000 грн в этом месяце» — прогресс-карта теряет смысл.
- **Решение:** компактный формат от 1 млн (`89,96 трлн ₴` + полное значение в `title`), `overflow-wrap`/auto-shrink для stat-значений. После 2.2 такие числа станут редкостью, но «дорогой» актив (квартира 5 млн) — реальный кейс.
```js
function fmtCompact(n){ return Math.abs(n) >= 1e6 ? new Intl.NumberFormat("ru-RU",{notation:"compact",maximumFractionDigits:2}).format(n)+" ₴" : fmt(n); }
```

### 2.4 🟡 Low · Типографика/i18n · Склонение и сырые float в сообщениях
- «**1 активов**» (карта портфеля) → нужна функция склонения (1 актив / 2 актива / 5 активов).
- `sell_exceeds_holdings` отдаёт `held: 3.0999999999999996` — в тосте видно сырой float → округлять до 8 знаков перед показом.
```js
const plural = (n, [one, few, many]) => { const m10=n%10, m100=n%100; return m10===1&&m100!==11?one:(m10>=2&&m10<=4&&(m100<10||m100>=20)?few:many); };
```

### 2.5 🟡 Low · Брендинг · Смешение алфавитов на экране входа
- **Где:** `index.html:86` — бейдж «ЛИЧНЫЙ ФИНАНСОВЫЙ COCKPIT»; капсом латиница читается как кириллическая абракадабра («СОСКРIТ»). На Кабинете при этом «личный финансовый штаб».
- **Решение:** один вариант везде — «Личный финансовый штаб» (раз русификация — линия продукта).

### 2.6 🟡 Low · UX · Поле даты в быстром добавлении — mm/dd/yyyy
- Нативный `input[type=date]` показывает формат локали браузера; для русскоязычного владельца с en-локалью браузера это ловушка перепутанных дня/месяца. Решение: подпись-плейсхолдер «дд.мм.гггг» рядом или явный hint, как у денежных полей.

### 2.7 🟡 Low · UX · Мигание пустой очереди при первом рендере
- Список желаний появляется после асинхронного `refresh()`; на медленном соединении секунду видна шапка с фильтрами без контента. Решение: skeleton-строки или «Загружаю…» в `#i-list`.

### 2.8 🟠 Medium (перенос) · Инфраструктура · Keepalive-workflow всё ещё не активен
